Today I finally did something I should have done on Day 1: I pushed my entire AI operations system to GitHub.

338 files. Every agent, every skill, every config, every course lesson — backed up in the cloud.

Here's what I learned doing it:

**1. .gitignore goes first. Always.**
Before any file touches GitHub, you protect your secrets. One accidental commit of an API key can compromise your entire system. The .gitignore is the lock on the door.

**2. Remote triggers and local cron are not the same thing.**
A remote trigger runs in Anthropic's cloud and can't see your local files. If your task reads files on your laptop — use a local cron. I almost set up a remote daily-report trigger that would have read nothing and failed silently.

**3. One project, one repo.**
My website was nested inside my ops project. That's a common mistake. They need to be separate — different purpose, different audience, different backup schedule.

**4. Git identity is a 30-second setup that most beginners skip.**
Your name and email on every commit. Set it once. Never think about it again.

**5. Skills vs identity.**
My CTO agent now IS a GitHub specialist — not because I tell it every session, but because it's part of who it is. If you have to remember to invoke expertise, you designed the agent wrong.
